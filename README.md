## 🚀 Overview
**Stock Sage** is designed to address a critical challenge in fresh produce retail: minimizing stock wastage while protecting profit margins. By monitoring expiry dates in real time, the application dynamically categorizes inventory based on expiration urgency and suggests smart markdown pricing.

### Key Capabilities:
- **Visual Urgency Tracking:** Automatically updates the background color of product cards based on remaining shelf life.
- **Dynamic Price Optimization:** Recommends markdown sale prices using custom discount rules but guarantees suggestions never drop below cost price.
- **Automated Sales Counting:** Decreasing inventory quantities via the interactive card sliders automatically reflects an increment in units sold.
- **Instant Profit Assessment:** Calculates real-time financial margins for every product based on sales volume and price deltas.
- **Persistent Local State:** Keeps inventory rosters, user accounts, and discount parameters saved across browser sessions using `localStorage`.

## 🧮 Helper Functions
The codebase leverages dedicated functional helpers to calculate date differentials, price recommendations, and financial results cleanly.

### `getDaysLeft(expiry)`
- **Purpose:** Calculates the exact integer count of full calendar days remaining before an item reaches expiration.
- **Operation:** Evaluates the millisecond delta between the user's localized system timestamp (`new Date()`) and the targeted entry date (`expiry`), transforming it into a normalized day count rounded up to the nearest integer.

### `suggestPrice(p, discounts)`
- **Purpose:** Dynamically determines a smart promotional sale price depending on how close an item is to its expiration.
- **Operation:** Calls `getDaysLeft()`. Based on matching predefined time brackets (e.g., 1–2 days, 2–3 days, or 3–5 days left), it applies the store's configured discount percentage against the original sale price. It executes a `Math.max()` constraint against the item's `costPrice` to ensure the retail recommendation never generates a net loss.

### `calculateProfit(p)`
- **Purpose:** Outputs an inline monetary calculation of current gross profit margins generated by an item line.
- **Formula:** $$	ext{Profit} = (	ext{Sale Price} - 	ext{Cost Price}) 	imes 	ext{Units Sold}$$
- **Operation:** Normalizes fields to standard numbers and outputs a rounded numeric text format set explicitly to two decimal places.

## 🛠️ Event Handlers & State Management
User actions trigger state modifications governed by robust event lifecycle handlers:

### Authentication Handlers
- **`handleLogin(e)`**: Intercepts form submission. Trims user input, conducts a case-insensitive lookup on the user state array, and verifies matching passwords. Grants access or sets a clear user warning notice on failure.
- **`handleLogout()`**: Terminates the active user session state, strips current user details, and explicitly closes administrative configuration panels.

### System Settings Handlers
- **`saveDiscounts()`**: Quantifies raw textual numeric percentage inputs out of the configuration layout fields and updates state records to represent decimal fraction modifiers.
- **`changePassword()`**: Authenticates the active operator's existing password against saved states, ensures confirmation entry fields match perfectly, prevents blank submission errors, and commits updates.
- **`addUser()`**: Disallows blank fields, performs validation checks against existing database arrays to prevent duplicate naming collusions, and appends a new user profile object.

### Inventory Grid Handlers
- **`updateField(i, field, value)`**: A flexible multi-purpose update method mapped to text, dropdown, and calendar inputs. Ensures values like stock quantities or price structures remain $\geq 0$. Crucially, if stock fields are manually typed downward, it identifies the change differential and increases the `sold` column tracking tally symmetrically.
- **`handleSliderChange(i, newStock)`**: Specially wired to the range slider component. When a store clerk glides the slider to reflect a reduction in shelf stock, this handler interprets the drop as immediate sales, scaling up the corresponding product `sold` metric automatically.
- **`addProduct()`**: Inserts a blank template object into the active inventory state array to initialize an empty product card row on screen.
- **`removeProduct(i)`**: Strips a targeted product entry cleanly out of the tracking list state based on its structural tracking element placement.

## 🔍 Sorting Architecture & Visual Logic
The system features an automated, index-preserving layout system that dynamically filters layout visualization rows while guaranteeing field updates modify accurate source arrays.

### Sorting Implementation
The application holds a standalone `sortBy` string state (`'default'`, `'alpha'`, or `'expiry'`). Rather than rearranging the primary immutable source state array directly—which would corrupt indices and cross-link edits to incorrect products—it computes a transient execution matrix named `sortedProducts`:
1. It maps the state array into temporary structural references that explicitly encapsulate their true initial array pointer (`_originalIndex`).
2. **Alphabetical View (`'alpha'`)**: Executes local comparisons using `.localeCompare()` against string elements in `product.name`.
3. **Expiration Urgency View (`'expiry'`)**: Evaluates integer return calculations via `getDaysLeft()`. Items closest to expiring (or past due) bubble directly to the top. Missing expiry variables fallback to an infinite value constraint to drop gracefully to the tail end of the list.

### 🎨 Visual Urgency Engine
Card UI presentations alter their background dynamically to guide clerk priorities through `getColor(days)` mapping variables:
* **🔴 Vibrant Red (`#ff3b30`)**: Critical urgency. Item has $\leq 1$ days remaining before loss.
* **🟡 Sunny Gold (`#ffcc00`)**: Caution. Item has between 2 and 3 days remaining.
* **🟢 Fresh Green (`#34c759`)**: Stable. Item has $> 3$ days of remaining shelf life.
